# Встановлення Docker Desktop

## Навіщо встановлювати Docker

Щоб запускати контейнери на своєму комп'ютері, потрібен **Docker**. **Docker Desktop** — це варіант для звичайних користувачів: він включає Docker Engine (рушій контейнерів), зручний інтерфейс і інтеграцію з ОС. Після встановлення ви зможете виконувати команди `docker` у терміналі та запускати тести в контейнерах локально так само, як це робить CI/CD.

**Передумова:** переконайтеся, що ваша ОС підтримується (Windows 10/11, macOS з підтримкою Apple Silicon або Intel). Офіційні вимоги завжди можна перевірити на [docker.com](https://docker.com).

---

## Крок 1. Завантаження

1. Відкрийте офіційний сайт: **https://www.docker.com**
2. Знайдіть розділ завантаження (наприклад, «Get Docker» / «Download Docker Desktop»).
3. Виберіть версію під **вашу ОС**:
   - **Windows** — Docker Desktop for Windows
   - **macOS** — Docker Desktop for Mac (обрати архітектуру: Apple Silicon або Intel)
4. Завантажте інсталяційний файл і дочекайтеся завершення завантаження.

---

## Крок 2. Встановлення

1. Запустіть завантажений інсталятор (подвійний клік по файлу).
2. Дотримуйтесь кроків майстра встановлення:
   - прийміть умови ліцензії, якщо це потрібно;
   - підтвердіть дозволи адміністратора, якщо система запитає.
3. **Windows:** коли з'явиться екран налаштувань, рекомендується увімкнути опцію **«Use WSL 2 instead of Hyper-V»** (використовувати WSL 2 замість Hyper-V), якщо вона доступна. WSL 2 дає кращу сумісність і продуктивність Docker на Windows.
4. Завершіть встановлення і за потреби **перезавантажте комп'ютер** (особливо на Windows після першого встановлення).

---

## Крок 3. Перший запуск і налаштування

1. Запустіть **Docker Desktop** (ярлик на робочому столі або через меню програм).
2. **Windows:** може з'явитися вікно з пропозицією увійти в Docker Hub або створити обліковий запис.  
   - Якщо у вас ще немає акаунта — нижче зазвичай є посилання на інструкцію з реєстрації. Реєстрація безкоштовна і потрібна для завантаження образів з Docker Hub (наприклад, офіційних базових образів). Без акаунта можна працювати локально, але публічні образи з Docker Hub можуть вимагати логін при pull.
3. **macOS:** для швидкого пошуку програми можна натиснути **Cmd+Space** і ввести `Docker`.
4. У налаштуваннях Docker Desktop можна виставити:
   - кількість виділеної **RAM** і **CPU**;
   - спільні папки між хостом і контейнерами (якщо потрібно).  
   Для початку підійдуть значення за замовчуванням.

Після запуску в треї (Windows) або в меню (macOS) з'явиться іконка Docker; коли вона перестане «крутитися», Docker готовий до роботи.

---

## Крок 4. Перевірка встановлення

1. Відкрийте **термінал** (Command Prompt, PowerShell або Terminal).
2. Виконайте команду:
   ```bash
   docker --version
   ```
   У відповідь має з'явитися номер версії (наприклад, `Docker version 24.0.x`).
3. Додатково перевірте роботу Docker запуском тестового контейнера:
   ```bash
   docker run hello-world
   ```
   Docker завантажить невеликий офіційний образ і виведе повідомлення про успішний запуск. Якщо це відбулося — встановлення пройшло коректно.

---

## Можливі відмінності

- Кроки та діалоги можуть трохи відрізнятися залежно від **версії ОС** та **версії Docker Desktop**.
- На **Windows** після оновлень іноді потрібне повторне перезавантаження або перезапуск Docker Desktop.
- Якщо виникають помилки при запуску контейнерів, перевірте офіційну документацію з встановлення для вашої платформи: [Docker Docs — Install](https://docs.docker.com/get-docker/).

---

## Що далі

Після успішної перевірки ви можете:

- використовувати команди `docker run`, `docker build` тощо для запуску тестів у контейнерах;
- налаштовувати образи та Docker Compose згідно з матеріалами в інших розділах (Dockerfile, образи, Cypress/Playwright у Docker).
